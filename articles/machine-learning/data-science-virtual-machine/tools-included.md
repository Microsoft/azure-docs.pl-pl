# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Jakie narzędzia są dostępne na platformie Azure Data Science Virtual Machine?

Data Science Virtual Machine to prosty sposób eksplorowania danych i uczenia maszynowego w chmurze. Virtual Machines analizy danych są wstępnie skonfigurowane z pełnymi systemami operacyjnymi, poprawkami zabezpieczeń, sterownikami i popularnym oprogramowaniem do nauki i programowania danych. Można wybrać środowisko sprzętowe, w odróżnieniu od tańszych od oszczędnych maszyn zorientowanych na procesor, do bardzo zaawansowanych maszyn z wieloma procesorami GPU, magazynu interfejsu NVMe i dużą ilością pamięci. W przypadku maszyn z procesorami GPU wszystkie sterowniki są instalowane. wszystkie platformy uczenia maszynowego są zgodne z wersją zgodną z procesorem GPU i przyspieszenie jest włączone we wszystkich programach aplikacji obsługujących procesory GPU.

Data Science Virtual Machine zawiera najbardziej przydatne narzędzia do nauki o danych, które są wstępnie zainstalowane.

## <a name="how-to-choosecondaenvironment"></a>Jak wybrać środowisko Conda  

> [!NOTE]
> Podczas przełączania do nowej wersji Data Science Virtual Machine ze starej wersji istnieją istotne zmiany. 

**Przełączanie z systemu Windows Server 2016 do systemu Windows Server 2019** </br> 

* Środowisko Conda jest ustawione na python37_default. Nie obsługujemy python2. Jeśli używasz TensorFlow lub pytorch w środowisku Python Conda, użyj odpowiednio py37_tensorflow lub py37_pytorch. Jeśli używasz tylko azureml_py36_automl usługi Azure use </br>

* Jeśli używasz TensorFlow lub pytorch w środowisku usługi Azure Conda, użyj odpowiednio azureml_py36_tensorflow lub azureml_py36_pytorch. </br>

**Przełączanie z Ubuntu 16,04 do Ubuntu 18,04** </br>

* Jeśli używasz TensorFlow lub pytorch w środowisku azureml_py36 Conda, użyj odpowiednio azureml_py36_tensorflow lub azureml_py36_pytorch. </br>

* Jeśli używasz TensorFlow lub pytorch w środowisku Conda py37, użyj odpowiednio py37_tensorflow lub py37_pytorch. </br>

* W przeciwnym razie użyj python37_default lub jeśli używasz tylko azureml_py36 użycia azureml_py36_automl </br>

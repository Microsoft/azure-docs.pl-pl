# <a name="team-data-science-process-project-planning"></a>Planowanie projektu analizy danych zespołu

Zespół ds. analizy danych (przetwarzania TDSP) zapewnia cykl życia do tworzenia projektów analizy danych. Ten artykuł zawiera linki do szablonów programu Microsoft Project i programu Excel, które ułatwiają planowanie tych etapów projektu i zarządzanie nimi.

Cykl życia przedstawia główne etapy, które są zwykle wykonywane przez projekty, często iteracyjnie:

- Zrozumienie biznesowe
- Pozyskiwanie i zrozumienie danych
- Modelowanie
- Wdrożenie
- Akceptacja klienta

Opisy poszczególnych etapów znajdują się w temacie [cykle procesu nauki o danych zespołowych](https://docs.microsoft.com/azure/machine-learning/team-data-science-process/lifecycle).

 
## <a name="microsoft-project-template"></a>Szablon programu Microsoft Project

Szablon programu Microsoft Project dla procesu nauka danych zespołu jest dostępny w tym miejscu: [szablon programu Microsoft Project](https://github.com/Azure/Azure-MachineLearning-DataScience/blob/master/Team-Data-Science-Process/Project-Planning-and-Governance/Advanced%20Analytics%20Microsoft%20Project%20Plan.mpp) 

Po otwarciu planu kliknij link po lewej stronie dla przetwarzania TDSP. Zmień nazwę i opis, a następnie Dodaj inne potrzebne zasoby zespołu. Oszacuj daty wymagane z Twojego środowiska.

![1](./media/team-data-science-process-project-templates/ms-project-templates.png)

Każde zadanie zawiera notatkę. Otwórz te zadania, aby zobaczyć, jakie zasoby zostały już utworzone.

![2](./media/team-data-science-process-project-templates/ms-project-template-task.png)


## <a name="excel-template"></a>Szablon programu Excel

Jeśli nie masz dostępu do programu Microsoft Project, arkusz programu Excel ze wszystkimi tymi samymi danymi jest również dostępny do pobrania tutaj: [szablon programu Excel](https://github.com/Azure/Azure-MachineLearning-DataScience/blob/master/Team-Data-Science-Process/Project-Planning-and-Governance/Advanced%20Analytics%20Microsoft%20Project%20Plan.xlsx) można ściągnąć do dowolnego narzędzia, którego wolisz użyć.

Używaj tych szablonów na własne ryzyko. Stosowane są [zwykle odrzuty](https://www.gnu.org/licenses/gpl-3.0.en.html) .

## <a name="repository-template"></a>Szablon repozytorium
